1. TECH LEAD - PENGENALAN UMUM PROJECT
==============================================================================

Assalamu'alaikum, selamat siang teman-teman. Nama saya [NAMA], sebagai Tech Lead 
dalam project ini.

Hari ini saya akan memperkenalkan project kami yang bernama "RENTAL APLIKASI" - 
sebuah sistem aplikasi web untuk manajemen penyewaan barang secara digital.

VISI PROJECT:
Aplikasi ini dirancang untuk memudahkan proses penyewaan barang dengan sistem yang 
terstruktur dan mudah digunakan oleh pengguna umum maupun admin.

FITUR-FITUR UTAMA:
1. Sistem Autentikasi (Login & Register)
2. Dashboard User untuk melihat riwayat penyewaan
3. Dashboard Admin untuk manajemen barang dan pengguna
4. Sistem Peminjaman dan Pengembalian
5. Manajemen Inventori Barang

TEKNOLOGI YANG DIGUNAKAN:
- Frontend: HTML, CSS, JavaScript
- Backend: PHP
- Database: MySQL
- Architecture: Object-Oriented Programming (OOP)

PROJECT STRUCTURE:
- Folder 'classes/' untuk class-class backend (Database, User, Item, Rental)
- Folder 'admin/' untuk halaman admin
- Folder 'user/' untuk halaman user
- Folder 'assets/' untuk CSS dan JavaScript
- Folder 'includes/' untuk file konfigurasi

Tim kami terdiri dari 4 orang dengan spesialisasi yang berbeda:
- 2 orang Front-End Developer menangani UI/UX dan interface
- 1 orang Back-End Developer menangani logika sistem dan database
- Saya sebagai Tech Lead mengkoordinasikan dan memastikan integrasi sempurna

Sekarang, saya akan memberikan kesempatan kepada rekan-rekan kami untuk menjelaskan 
bagian masing-masing. Mari kita mulai dengan Front-End 1.

4. BACK-END DEVELOPER - SISTEM DATABASE & BACKEND
==============================================================================

Terima kasih. Nama saya [NAMA], saya adalah Back-End Developer dalam project ini.

TANGGUNG JAWAB SAYA:
Saya menangani logika bisnis, sistem autentikasi, database, dan semua proses 
backend yang mendukung aplikasi ini.

STRUKTUR CLASS BACKEND:

1. DATABASE CLASS (classes/Database.php)
   - Mengelola koneksi ke database MySQL
   - Query builder untuk insert, update, delete, select
   - Prepared statement untuk keamanan SQL Injection
   - Error handling dan logging
   - Connection pooling jika diperlukan

2. USER CLASS (classes/User.php)
   - Register pengguna baru dengan validasi
   - Login dengan verifikasi password (hashing)
   - Mendapatkan informasi user dari database
   - Update profil pengguna
   - Ubah password dengan verifikasi password lama
   - Session management

3. ITEM CLASS (classes/Item.php)
   - CRUD operation untuk barang
   - Tambah barang baru ke inventori
   - Update informasi barang
   - Delete barang (soft delete)
   - Tampilkan daftar barang
   - Cek ketersediaan barang

4. RENTAL CLASS (classes/Rental.php)
   - Create rental/penyewaan baru
   - Update status penyewaan
   - Track tanggal peminjaman dan pengembalian
   - Hitung biaya rental
   - Histori penyewaan pengguna
   - Late fee calculation jika ada keterlambatan

DATABASE SCHEMA:

TABLE: users
- id (INT, Primary Key)
- nama (VARCHAR)
- email (VARCHAR, UNIQUE)
- password (VARCHAR, hashed)
- telepon (VARCHAR)
- alamat (TEXT)
- tanggal_registrasi (TIMESTAMP)
- status (ENUM: active, inactive)

TABLE: items
- id (INT, Primary Key)
- nama_barang (VARCHAR)
- deskripsi (TEXT)
- kategori (VARCHAR)
- stok (INT)
- harga_rental_per_hari (DECIMAL)
- kondisi (ENUM: baik, rusak ringan, rusak berat)
- tanggal_ditambahkan (TIMESTAMP)

TABLE: rentals
- id (INT, Primary Key)
- user_id (INT, Foreign Key)
- item_id (INT, Foreign Key)
- tanggal_peminjaman (DATE)
- tanggal_pengembalian_rencana (DATE)
- tanggal_pengembalian_aktual (DATE, nullable)
- total_harga (DECIMAL)
- status (ENUM: pending, approved, rejected, returned, overdue)

FITUR-FITUR BACKEND:

1. AUTENTIKASI & SECURITY
   - Password hashing menggunakan bcrypt atau password_hash()
   - Session management yang aman
   - CSRF protection pada form
   - Input validation dan sanitization
   - SQL Injection prevention dengan prepared statement

2. LOGIKA BISNIS
   - Validasi ketersediaan barang sebelum rental
   - Kalkulasi biaya berdasarkan durasi
   - Auto-update status rental
   - Late fee handling
   - Notification system

3. ERROR HANDLING
   - Try-catch untuk exception handling
   - Logging ke file untuk debugging
   - User-friendly error messages
   - Database error management

4. API ENDPOINTS (jika ada)
   - GET /api/items - Daftar barang
   - POST /api/rentals - Buat rental baru
   - GET /api/user/rentals - Riwayat rental user
   - POST /api/confirm-return - Konfirmasi pengembalian

FILE KONFIGURASI (includes/config.php):
- Database connection settings
- Global constants
- Helper functions
- Session initialization

PROSES ALUR SISTEM:

1. REGISTRASI USER
   - Input username, email, password
   - Backend validasi uniqueness email
   - Hash password
   - Simpan ke database
   - Redirect ke login page

2. LOGIN USER
   - Input email & password
   - Query database cari user by email
   - Verify password dengan hash
   - Create session jika valid
   - Redirect ke dashboard

3. PROSES PENYEWAAN
   - User pilih barang
   - Input durasi penyewaan
   - System cek stok barang
   - Kalkulasi total harga
   - Create rental record
   - Kirim notifikasi ke admin
   - Admin confirm/reject
   - Update status

4. PENGEMBALIAN BARANG
   - User request return
   - Admin confirm return
   - Cek kondisi barang
   - Update rental status menjadi 'returned'
   - Update stok barang
   - Hitung late fee jika ada

SECURITY MEASURES:
- Password hashing dengan algorithm yang aman
- Prepared statements untuk prevent SQL Injection
- Input validation di backend (bukan hanya frontend)
- Session timeout untuk security
- Authorization check sebelum akses data

Dengan sistem backend yang robust ini, aplikasi kami dapat menangani logika bisnis 
yang kompleks dengan aman dan efisien.
